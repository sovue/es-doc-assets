# Аудио в Ren'Py
## Принимаемые форматы файлов
Чаще всего используются:
- WAV — Высокое качество с минимальными потерями и большими размерами файлов, но Ren'Py поддерживает только один формат файла — несжатый подписанный 16-битный PCM (signed 16-bit PCM). Рекомендуется для бесшовного повтора (когда момент повтора не заметен слушателю). РМШ — 5 169 КБ.
- MP3 — Файлы сжимаются до минимально возможного размера с сохранением качества, но в большем количестве звуковых редакторов нельзя экспортировать файл без пауз в начале и конце. Не рекомендуется для бесшовного повтора. РМШ — 2 346 КБ.
- Ogg Vorbis — Альтернатива MP3 с открытым кодом, настолько же хорошим качeством аудио и меньшим, по сравнению с MP3, размером файла. Рекомендуется для бесшовного повтора. РМШ — 1 852 КБ.
Непопулярные:
- FLAC — Формат, предназначенный для сжатия аудио **без потерь**, но имеющий самый большой размер файлов из всех возможных для проигрывания в Ren'Py. РМШ — 7 695 КБ.
- Opus — Формат, все преимущества которого (кроме сжатия) не важны для разработки мода. Более эффективное сжатие файлов с идентичными потерями как у MP3. РМШ — 1 759 КБ.
- MP2 — Формат, используемый до MP3. Хуже его в уровне сжатия и качестве аудио. РМШ — 2 811 КБ.

> Под бесшовным повтором имеется ввиду повторное воспроизведение файла, с незаметным переходом в начало воспроизведения, и ощущением непрерывности с самого начала.

> Под РМШ имеется ввиду примерный размер файла, с одной и той же записью белого шума, длительностью ровно в одну минуту и с наилучшим возможным качеством файла, в килобайтах. Конвертация в другие форматы производилась в программе Audacity, с кодеком LAV Audio Decoder.

---

По этим данным можно составить таблицу:

:::table
Формат;Качество;Размер;РМШ (КБ)
WAV;9/10;3/10;5 169
MP3;6/10;7/10;2 346
OGG Vorbis;8/10;9/10;1 852
FLAC;10/10;2/10;7 695
Opus;6/10;10/10;1 759
MP2;5/10;5/10;2 811
:::

Таким образом можно сделать вывод что:
- Лучшее соотношение качества и размера файла имеет формат OGG Vorbis. Он подходит лучше всего, если вы не хотите тратить место и иметь адекватное качество аудио.
- Лучшее качество звука имеет формат FLAC. Он не имеет потерь, но и весит больше всех.
- Меньше всего места занимает формат Opus. Имея то же качество, что и MP3, весит он почти в полтора раза меньше.
- Если вам не важно качество, размер файлов и бесшовность, то вы можете использовать формат MP3, с его адекватным качеством звука, размерами файлов и распространённостью.

## Аудиоканалы
Ren'Py поддерживает неограниченное количество аудиоканалов. По умолчанию в Ren'Py созданы всего три:
- `music` — канал для проигрывания музыки. Аудиофайлы, проигрываемые на этом канале, по умолчанию повторяются.
- `sound` — канал для проигрывания звуковых эффектов. Аудиофайлы, проигрываемые на этом канале, по умолчанию не повторяются.
- `voice` — канал для проигрывания голоса персонажей. Аудиофайлы, проигрываемые на этом канале, по умолчанию не повторяются.

При разработке Бесконечного Лета были созданы ещё несколько каналов:
- `sound2`, `sound3` — аналогичны каналу `sound`.
- `sound_loop`, `sound_loop2`, `sound_loop3` — каналы для проигрывания повторяющихся звуков. Аудиофайлы, проигрываемые на этом канале, по умолчанию повторяются.
- `ambience` — канал для проигрывания эмбиента (фонового шума). Аудиофайлы, проигрываемые на этом канале, по умолчанию повторяются.
Помимо Обычных аудиоканалов существует один специальный — `audio`. Этот канал поддерживает одновременное воспроизведение нескольких аудиофайлов, но не поддерживает использование команд `queue` и `stop` (несмотря на это, звуки в этом канале всё ещё можно воспроизводить с зацикливанием (аргумент `loop`), что может очень испортить игровой процесс). Также этот аудиоканал используют свойства стиля `hover_sound` и `activate_sound`.

## Оператор `play`
Оператор `play` используется для воспроизведения аудиофайла. Если файл в данный момент воспроизводится на обычном канале, он прерывается и заменяется новым файлом.
После оператора `play` ожидается название аудиоканала. За ним следует путь к файлу или переменная, содержащая путь к файлу.
```renpy
play назвКанала "путь/к/файлу"
play назвКанала переменная
```

Примеры:
```renpy
play sound "mymod/sounds/explosion.ogg"
play mymod_channel_ambienceAddition mymod_sfx_addition_scream
```

Вместо файла можно использовать список файлов, тогда его элементы будут воспроизводиться по порядку.
```renpy
play назвКанала [переменная, переменная2, переменная3]
```

Пример:
```renpy
play music [music_list["everyday_theme"], music_list["pile"], music_list["lets_be_friends"]]
```

---

### Аргументы `fadein`, `fadeout`
Аргумент `fadeout` создает эффект нарастания громкости звука, а `fadein` — его затухания. Если аргумент `fadeout` не задан, для затухания используется значение конфигурационной переменной `config.fadeout_audio` (По умолчанию равен `0.0`).
```renpy
play назвКанала переменная fadeout времяЗатухания fadein времяНарастания
```

Пример:
```renpy
play music music_list["orchid"] fadein 5.0 fadeout 2.5
```

---

### Аргумент `loop`
Аргумент `loop` позволяет зацикливать звук при его проигрывании (допустим если звук проигрывается в канале без зацикливания и использован аргумент `loop`, то он будет повторяться).
```renpy
play назвКанала переменная loop
```

Пример:
```renpy
play sound sfx_achievement loop
```

---

### Аргумент `noloop`
Аргумент `noloop` позволяет проигрывать звук без зацикливания при его проигрывании (допустим если звук проигрывается в канале с зацикливанием и использован аргумент `noloop`, то повторяться он не будет).
```renpy
play назвКанала переменная noloop
```

Пример:
```renpy
play music sfx_achievement noloop
```

---

### Аргумент `if_changed`
С аргументом `if_changed` звук на канале начнёт проигрываться только если он не играл до этого.
```renpy
play назвКанала переменная if_changed
```

Пример:
```renpy
    play music music_list["forest_maiden"]
    jump next_label
label next_label:
    play music music_list["forest_maiden"] if_changed
```

---

### Аргумент `volume`
Аргумент `volume` позволяет настроить относительную громкость проигрываемого звука в диапазоне от `0.0` до `1.0`, где `0.0` — самая низкая громкость, `1.0` — самая высокая громкость. (Можно использовать значения и больше `1.0`, но это крайне не рекомендуется. **Будет больно.**)
```renpy
play назвКанала переменная volume громкость
```

Пример:
```renpy
play sound sfx_bush_leaves volume 0.1
```

:::warning
**Внимание! Функция работает только с версии Ren'Py 7.4.0 и выше, из-за этого она не будет работать в старой версии Бесконечного Лета (бета — 7.3.5)**
:::

## Оператор `stop`
Оператор `stop` используется для остановки воспроизведения аудиофайла. В начале пишется ключевое слово `stop`, за которым следует имя аудиоканала, на котором необходимо остановить файл. Также этот оператор может иметь аргумент `fadeout`. Если аргумент `fadeout` не задан, для затухания используется значение конфигурационной переменной `config.fadeout_audio`.
```renpy
stop назвКанала
stop назвКанала fadeout времяЗатухания
```

Примеры:
```renpy
stop music
stop ambience fadeout 2.0
```

## Оператор `queue`
Оператор `queue` используется для постановки аудиофайлов в очередь. Они будут воспроизведены, когда канал завершит воспроизведение текущего воспроизводимого файла.
```renpy
queue назвКанала переменная
```

В начале пишется ключевое слово `queue`, за которым следует имя канала, на котором будет воспроизводиться звук. Аргументы `fadein`, `loop`, `noloop`, `volume` — опциональны, и работают также как и в операторе `play`.

Пример:
```renpy
play music "music/intro.ogg" noloop
queue music "music/loop.ogg"
```

## Дополнительные манипуляции с аудио
### Частичное воспроизведение
Ren'Py поддерживает частичное воспроизведение аудиофайлов. Это делается путем помещения в угловые скобки уточнения, откуда и докуда должен проигрываться файл, перед самим файлом. Значения всегда интерпретируются как секунды от начала файла.
```renpy
play назвКанала "<from началоЧасти to конецЧасти>" + переменная
play назвКанала "<loop началоПовтора>" + переменная
```

`началоЧасти` указывает позицию в файле, с которой начинается первое воспроизведение. По умолчанию это значение равно `0.0` секунд.

`конецЧасти` указывает позицию в файле, на которой заканчивается воспроизведение файла. По умолчанию это **полная продолжительность файла**.

`началоПовтора` указывает позицию в файле, с которой начинается воспроизведение второго и последующих прохождений. По умолчанию это время начала, заданное параметром `from`, если указано, или `0.0` секунд.

Примеры:
```renpy
play music "<from 5 to 15.5>mymod/music.ogg"
```
```renpy
play music "<loop 6.333>" + mymod_music
```
```renpy
play music "<from 0 to 10>" + music_list["smooth_machine"]
```

---

### Синхронизация позиции старта
Позицию в файле, с которой начинается воспроизведение клипа, можно синхронизировать с другим каналом с воспроизводимой в данный момент дорожкой, используя выражение `sync`.
```renpy
play назвКанала "<sync назвКанала&gt;" + переменная
```

Пример:
```renpy
play music "layer1.ogg"
pause
play music2 "<sync music>layer2.ogg"
```

:::warning
**Внимание! Функция работает только с версии Ren'Py 7.4.5 и выше, из-за этого она не будет работать в Старой версии Бесконечного Лета (бета — 7.3.5)**
:::

---

### Проигрывание тишины
В Ren'Py имеется возможность проигрывания тишины. Для этого вместо файла впишите:
```renpy
"<silence длительность>"
```

Пример:
```renpy
play music "<silence 1.5>" noloop
queue music "mymod/music.ogg"
```

## Python-функции связанные с аудио
### Управление воспроизведением
```python
renpy.play(filename, channel=None, **kwargs)
```

Проигрывает звук `filename` на канале `channel`. Если аргумент `channel` имеет значение `None`, то он будет иметь значение конфигурационной переменной `config.play_channel`.

---

```python
renpy.music.play(filenames, channel='music', loop=None, fadeout=None, synchro_start=False, fadein=0, tight=None, if_changed=False, relative_volume=1.0)
```

- `filenames` — файл или список файлов для проигрывания.
- `channel` — аудиоканал на котором проиграется музыка. По умолчанию — `music`.
- `loop` — если значение этого аргумента равно `True`, то воспроизведённый файл будет повторяться. По умолчанию — `None`.
- `fadein` — если значение не равно `None`, то время, за которое громкость файла будет нарастать.
- `fadeout` — если значение не равно `None`, то время, за которое громкость файла будет затухать.
- `synchro_start` — если значение равно `True`, то файл будет синхронизировано воспроизведён вместе с другими файлами, запущенными через эту функцию с этим аргументом равным `True`.
- `tight` — если значение равно `True`, то затухание будет распространяться и на следующий файл в очереди. Если `None`, то используется значение аргумента `loop`.
- `if_changed` — если `True` и на этом канале уже воспроизводится файл, то его воспроизведение не будет прервано.
- `relative_volume` — устанавливает громкость файла относительно громкости микшера в диапазоне от `0.0` до `1.0`, где `0.0` — минимальная громкость, `1.0` — максимальная громкость.

---

```python
renpy.music.queue(filenames, channel='music', loop=None, clear_queue=True, fadein=0, tight=None, relative_volume=1.0)
```

- `filenames` — файл или список файлов для проигрывания.
- `channel` — аудиоканал на котором проиграется музыка. По умолчанию — `music`.
- `loop` — если значение этого аргумента равно `True`, то воспроизведённый файл будет повторяться. По умолчанию — `None`.
- `clear_queue` — если `True`, то очередь будет очищена, и файл сразу начнёт своё воспроизведение. **(Внимание! По умолчанию этот аргумент всегда равен** `True`**!)**
- `fadein` — если значение не равно `None`, то время, за которое громкость файла будет нарастать.
- `tight` — если значение равно `True`, то затухание будет распространяться и на следующий файл в очереди. Если значение равно `None`, то используется значение аргумента `loop`.
- `relative_volume` — устанавливает громкость файла относительно громкости микшера в диапазоне от `0.0` до `1.0`, где `0.0` — минимальная громкость, `1.0` — максимальная громкость.

---

```python
renpy.music.stop(channel='music', fadeout=None)
```

- `channel` — аудиоканал на котором будет прекращено воспроизведение файла. По умолчанию — `music`.
- `fadeout` — если значение не равно `None`, то время, за которое громкость файла будет затухать.

---

### Получение информации
```python
renpy.music.get_duration(channel='music')
```

Возвращает длительность файла, воспроизводимого на канале `channel`, в секундах. По умолчанию аргумент channel равен `'music'`.

---

```python
renpy.music.get_pos(channel='music')
```

Возвращает текущую позицию воспроизведения файла, воспроизводимого на канале `channel`, в секундах. По умолчанию аргумент channel равен `'music'`.

---

```python
renpy.music.get_loop(channel='music')
```

Возвращает `True`, если воспроизведение файла на канале `channel` зациклено. По умолчанию аргумент `channel` равен `'music'`.


:::warning
**Внимание! Функция работает только с версии Ren'Py 7.4.5 и выше, из-за этого она не будет работать в Старой версии Бесконечного Лета (бета — 7.3.5)**

:::

---

```python
renpy.music.get_pause(channel='music')
```

Возвращает `True`, если воспроизведение файла на канале `channel` приостановлено функцией `renpy.music.pause()` или экранным действием `Pause()`. По умолчанию аргумент `channel` равен `'music'`.

---

```python
renpy.music.is_playing(channel='music')
```

Возвращает `True`, если в данный момент на канале `channel` воспроизводится любой файл. По умолчанию аргумент channel равен `'music'`.

---

```python
renpy.music.get_playing(channel='music')
```

Возвращает путь к файлу, проигрываемому в данный момент на канале `channel`. По умолчанию аргумент channel равен `'music'`.

---

```python
renpy.seen_audio(filename)
```

Возвращает `True`, если файл `filename` хоть один раз был воспроизведён в системе или помечен как воспроизведённый функцией `renpy.mark_audio_seen(filename)`.

### Прочие

```python
renpy.music.register_channel(name, mixer, loop=None, stop_on_mute=True, tight=False, file_prefix='', file_suffix='')
```

Создаёт новый аудиоканал с именем `name`.
- `name` — название нового канала.
- `mixer` — микшер нового канала. Может быть `music`, `sound` или `voice`.
- `loop` — если значение этого аргумента равно `True`, то воспроизведённые на этом канале файлы будет повторяться. По умолчанию — `None`.
- `stop_on_mute` — если `True`, то воспроизведение файлов будет остановлено, если громкость микшера будет равна нулю. По умолчанию — `True`.
- `tight` — если `True`, то затухание будет распространяться и на следующий файл в очереди на этом канале. По умолчанию — `False`.
- `file_prefix` — устанавливает приставку в начале каждого файла, воспроизводимого на этом канале. По умолчанию — `''`.
- `file_suffix` — устанавливает приставку в конце каждого файла, воспроизводимого на этом канале. По умолчанию — `''`.

---

```python
renpy.music.set_pan(pan, delay, channel='music')
```

Настраивает панорамирование звука, используя значение аргумента `pan`, на аудиоканале `channel` в течение `delay` секунд. Значение аргумента `pan` должно находится в диапазоне от `-1.0` до `1.0`, где `-1.0` — весь звук в левом канале, `1.0` — весь звук в правом канале. По умолчанию значение аргумента channel равно `'music'`.

---

```python
renpy.music.set_pause(value, channel='music')
```

Если значение аргумента `value` равно `True`, то воспроизведение файлов на канале `channel` будет приостановлено. По умолчанию значение аргумента channel равно `'music'`.

---

```python
renpy.music.set_volume(volume, delay=0, channel='music')
```

Настраивает громкость звука на канале `channel`, используя значение аргумента `volume`, в течение `delay` секунд. Значение аргумента `volume` должно находится в диапазоне от `0.0` до `1.0`, где `0.0` — минимальная громкость, `1.0` — максимальная громкость. По умолчанию значение аргумента `delay` равно `0`, значение аргумента `channel` равно `'music'`.

---

```python
renpy.music.set_queue_empty_callback(callback, channel='music')
```

Устанавливает функцию `callback`, которая выполнится когда в списке воспроизведения канала channel закончатся файлы. По умолчанию значение аргумента channel равно `'music'`.

---

```python
renpy.mark_audio_seen(filename)
```

Помечает файл `filename` уже воспроизведённый в системе.

---

```python
renpy.mark_audio_unseen(filename)
```

Помечает файл `filename` как ни разу не воспроизведённый в системе.

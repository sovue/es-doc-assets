GitHub - это удобный инструмент для работы в большой команде, удобного резервного копирования и простого хранения модификации в облаке. Каждый член команды может скачать актуальную версию мода, добавить свои правки и это будет показано в истории.

---

Основные понятия, с которыми вы будете работать, это **репозиторий** и **коммит**. 

Репозиторий — это место, где хранятся и поддерживаются какие-либо данные. В нашем случае: файлы модификации.

Коммит — это операция, которая отправляет последние изменения исходного кода в репозиторий.

## Создание репозитория

Под созданием репозитория можно понять три вещи:

- Создание нового **пустого** репозитория;
- Создание нового репозитория **из папки мода**;


- **Клонирование** репозитория.

---

### Создание пустого репозитория

Для создания пустого репозитория зайдите в приложение GitHub Desktop, в верхнем левом углу окна откройте меню *File* и нажмите *New Repository…*.

 ![](/api/attachments.redirect?id=c8ca9c8b-ee3b-4571-8b35-5b22f326fce8 "aspect=1")

После этого в поле *Name* введите название своего мода (желательно вводить название папки с модом, латиницей и без спецсимволов, например `“ray_v_panelke“` или `“wnfh“`), в поле *Local path* введите путь к тому месту, где будет храниться ваш мод и находиться ваш локальный репозиторий (если вы хотите, чтобы в мод сразу можно было играть, то выберите путь, ведущий к папке `Steam/steamapps/common/Everlasting Summer/game` или `Steam/steamapps/workshop/content/331470/”любое название папки”`) и нажмите на кнопку *Create repository*.

 ![](/api/attachments.redirect?id=30ed5a41-a6ab-499c-87ba-71d504090962 "aspect=1")

Для того чтобы опубликовать ваш репозиторий, нажмите на кнопку *Publish repository* в главном окне и в окне публикации.

 ![](/api/attachments.redirect?id=3138b3eb-2714-4dcd-9363-c0d3d6d1db44 "aspect=1")

 ![](/api/attachments.redirect?id=3f4b538c-bb67-4412-b8a4-9a1ad382b839 "aspect=1")

### Создание репозитория из папки мода


:::warning
Перед этим обязательно выйдите из Бесконечного Лета!

:::

Для этого **временно переименуйте** папку своего мода (не обязательно полностью переписывать название, нужно чтобы оно как-либо отличалось от старого) и **создайте пустой репозиторий** в папке расположения папки вашего мода **с названием в виде имени папки вашего мода**. После этого перенесите все файлы из папки мода в папку репозитория и опубликуйте его как в гайде выше.

---

### Клонирование репозитория


:::warning
Для **клонирования репозитория** убедитесь, что у вас есть к нему **[доступ](#h-predostavlenie-dostupa-k-repozitoriyu)**.

:::

Для клонирования репозитория в верхнем левом углу окна откройте меню *File* и нажмите *Clone repository…*.

 ![](/api/attachments.redirect?id=88f6ad45-a038-4899-8a5a-e25828819d56 "aspect=1")

После этого выберите нужный вам репозиторий, в поле *Local path* введите путь к тому месту, где будет храниться ваш мод и находиться ваш локальный репозиторий (если вы хотите, чтобы в мод сразу можно было играть, то выберите путь, ведущий к папке `Steam/steamapps/common/Everlasting Summer/game` или `Steam/steamapps/workshop/content/331470/”любое название папки”`) и нажмите на кнопку *Clone*.

## Операции Fetch, Pull и Push

Основные типы операций над репозиторием — это Fetch, Pull и Push.

Fetch — операция получения актуальных данных о репозитории.

Эта операция доступна если у вас нет невыгруженных данных в локальном репозитории или если GitHub Desktop не видит изменений в онлайн-репозитории.

Pull — операция “подтягивания“ актуальных данных из онлайн-репозитория и скачивания их в локальный репозиторий (загрузки файлов из интернета).

Эта операция доступна если у вас нет невыгруженных данных в локальном репозитории и GitHub Desktop видит изменения в онлайн-репозитории, не представленные в локальном репозитории.

Push — операция “проталкивания“ данных из локального репозитория в онлайн-репозиторий (выгрузки файлов в интернет).

Эта операция доступна если у вас имеются невыгруженные данные в локальном репозитории и вы создали **коммит**. Для этого введите его название (можете кратко описать изменения), введите описание (более длинное описание изменений, необязательно) и нажмите *Commit to **main***.

 ![](/api/attachments.redirect?id=9ebe702e-d5db-496c-a49b-d637427b09ce "aspect=1")

По отдельности все эти операции доступны во вкладке Repository, а также допустимую операцию можно вызвать нажав на соответствующую кнопку справа от выбора ветки.

## .gitignore

Если в папке локального репозитория создать файл c названием *.gitignore*, то в онлайн-репозиторий не будут загружаться файлы, пути к которым записаны в этом файле. К примеру:

```none
*.rpyc
exactFile.txt
/folder1/folder2
/folder1/exactFile.txt
/folder1/*.mp3
```

Файл *.gitignore* с таким содержимым будет игнорировать все файлы *.rpyc*, файл *exactFile.txt*, всё содержимое вкладки *folder2* в папке *folder1*, файл *exactFile.txt* в папке *folder1* и все файлы *.mp3* в папке *folder1*.

Вы можете настроить свой *.gitignore* как хотите, выделяя нужные файлы для игнорирования репозиторием.

## Предоставление доступа к репозиторию

Чтобы предоставить доступ к своему репозиторию другим пользователям перейдите на **страницу вашего репозитория**, войдите в **настройки** и во вкладке *Collaborators* нажмите кнопку *Add people*.

 ![](/api/attachments.redirect?id=53285374-0474-4bee-9f04-a480634493e6 "aspect=1")

Введите никнейм пользователя или адрес электронной почты, выберите из списка нужного вам пользователя и нажмите *Add имяПользователя*. 

 ![](/api/attachments.redirect?id=6486b6a2-9c8f-42c4-9d6f-9f2c487a5f36 "aspect=1")

 ![](/api/attachments.redirect?id=9420c0b9-be3f-4a15-b794-14b0840fc8c3 "aspect=1")

В течение нескольких минут пользователю на электронную почту придёт письмо, со ссылкой на принятие предоставления доступа к репозиторию. С этого момента этот пользователь сможет [клонировать ваш репозиторий](#h-klonirovanie-repozitoriya) к себе на компьютер.


:::warning
Предоставлять доступ к репозиторию может только его создатель!

:::
